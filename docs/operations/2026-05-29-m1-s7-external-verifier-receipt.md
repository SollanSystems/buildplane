## Summary

M1-S7 ships the first **external** verification path for Buildplane's signed
ledger tape ‚Äî a consumer can independently confirm event-envelope hashes,
detached signatures, and the checkpoint Merkle root against trusted public keys
without the Buildplane runtime:

- **External verifier** (`scripts/verify-signed-tape.mjs`): a dependency-light
  Node script that reads a `buildplane.signed-tape.v1` tape, re-derives each
  event's canonical-byte hash, checks each detached signature against the tape's
  trusted public keys, recomputes the checkpoint Merkle root, and exits nonzero
  on any mismatch.
- **Reference fixture generator** (`native/crates/bp-ledger/src/bin/gen_signed_tape.rs`):
  a `bp-ledger` crate binary that emits deterministic `valid`, `tampered`, and
  `bad-root` signed-tape fixtures using the crate's own canonicalization,
  signing, and checkpoint logic ‚Äî so the fixtures are ground-truth, not
  hand-authored.
- **Checked-in fixtures** (`test/fixtures/signed-tape/{valid,tampered,bad-root}/tape.json`):
  byte-stable artifacts regenerated by `pnpm ledger:gen-fixtures`.
- **Vitest coverage** (`test/workflow/verify-signed-tape.test.ts` +
  additions to `test/workflow/ledger-doc-contract.test.ts`): asserts the verifier
  passes on `valid` and fails on the tamper/bad-root fixtures, and that the
  documented CLI contract in `docs/ledger.md` matches the script.
- **Operator docs** (`docs/ledger.md`): documents the `buildplane.signed-tape.v1`
  format, the verifier invocation, and the deferred live-DB export follow-up.
