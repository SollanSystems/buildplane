# M2-S4 — PlanForge Dispatch Stage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire the PlanForge `dispatch` stage so an operator-admitted plan spawns one run / `UnitPacket` per `PlanForgeTask` in an isolated worktree, gated on a prior signed `plan_admitted` tape event, with `provenance_ref` recorded on the packet + admission receipt and a *real* `worktree_clean` git status.

**Architecture:** A new `UnitPacket.provenance_ref` field carries the signed `plan_admitted` event id. The kernel `admitPreparedRunAsync` admission gate (the PlanForge dispatch path) refuses to dispatch any packet whose `provenance_ref` does not resolve to a kernel-signed `plan_admitted` event authorizing `dispatch_admitted_plan` — re-establishing dispatch trust from the durable tape, not the process-local `WeakSet`. The hardcoded `worktree_clean: true` is replaced by a live `git status --porcelain` against the isolated worktree (via a new `checkWorktreeClean` workspace-port method). A new `packages/planforge/src/dispatch.ts` builds packets from tasks; a new `buildplane planforge dispatch` CLI command pre-checks admission and runs each packet through `runPacketAsync` (sequential, respecting `dependsOn`).

**Tech Stack:** TypeScript (pnpm monorepo, `@buildplane/{kernel,planforge,adapters-git}` + `apps/cli`), vitest 4, `node:sqlite` `DatabaseSync` (read-only tape reads), Rust `bp-ledger` tape (read-only here — no new event kind).

## Resolved design decisions

Three load-bearing trust decisions were resolved with the operator before planning; the rest are engineering-judgment calls grounded in the survey. **Do not silently change these — they are the spec of this slice.**

| # | Decision | Resolution | Why |
|---|----------|-----------|-----|
| **D1** | Where is the "must be admitted" gate enforced? | **Kernel-enforced (defense-in-depth).** The CLI `planforge dispatch` pre-checks for a fast, clear failure **and** the kernel `admitPreparedRunAsync` enforces it (fail-closed `plan-not-admitted`). | Matches spec line 171 ("orchestrator.ts … gate dispatch on a prior plan_admitted") and lays the S7b crash-recovery groundwork (trust re-established from the tape). |
| **D2** | What does `provenance_ref` carry / how does the gate look it up? | **The `plan_admitted` event id.** Gate does a direct primary-key lookup `WHERE id = ? AND kind = 'plan_admitted'`. | A precise durable tape pointer; avoids the high-severity "wrong `run_id` partition" bug (no `planAdmitRunId` re-derivation at the gate). |
| **D3** | How deeply is the `plan_admitted` signature verified at the gate? | **Structural check** (`event_signatures` row has `actor_id='kernel'`, `key_id='kernel-main'`, `algorithm='ed25519'`). Full Ed25519 byte verification deferred to M3. | Matches the existing `readTape` pattern (`planforge-admit.test.ts:74`); `scripts/verify-signed-tape.mjs` remains the full-crypto path, run as a separate operator step. |
| **E1** | `parseUnitPacket` on missing `provenance_ref` | Field is typed **required** (`provenance_ref: string`); the parser **defaults a missing JSON field to `""`** (the legacy / non-PlanForge lane). A *present* value must be a non-empty string or it throws. The kernel gate only fires when `provenance_ref` is non-empty. | Honors the spec's "required" (every packet has the field) without breaking the non-PlanForge `buildplane run` callers and legacy stored packets that have no provenance. |
| **E2** | `worktree_clean` verification | Add a `checkWorktreeClean(worktreePath)` method to `BuildplaneWorkspacePort` + `adapters-git`, reusing the existing `.buildplane/**` (incl. `.buildplane/ledger/**` WAL) exclusion pathspecs. Fail-closed (returns `false` on any git error). | The exclusion logic already lives in `adapters-git`; a raw inline `execFileSync` in the orchestrator would duplicate it and risk WAL files reading as "dirty". |
| **E3** | Multi-task dispatch shape | **Sequential `runPacketAsync`** (PF1, then PF2 only if PF1 passes), not `runGraphAsync`. | The task count is fixed at two and PF2 always `dependsOn` PF1; the graph scheduler is unneeded surface for S4. |

## Out of scope / deferred (do NOT build in S4)

- **`run_admission_recorded` → signed-tape mirror.** The milestone constraint (spec lines 34, 85) that the JSONL-sidecar admission record be mirrored onto the signed tape is **not** in S4's file list or acceptance criteria, and adds a second signed-write path with its own atomicity semantics deserving a dedicated L0 slice. **Recommendation:** handle it as a tight follow-up (S4.5) or fold into S5. The existing JSONL sidecar write (`appendAdmissionEvent`, `orchestrator.ts:152`) stays untouched. _Flag this deferral for operator sign-off at the plan-review gate._
- **Real worker/model execution per task.** S4 dispatches a deterministic no-op command packet (`execution.command: "true"`) to prove the admission → run → worktree path. Binding a task's objective to a real model/worker run is **S5** (activity bracketing).
- **`bp-replay` transitions for `plan_admitted`.** Replay arms stay no-op (`transitions.rs`); that is **S7a**. S4 adds **no** new tape event kind, so the Rust workspace is a no-op regression check only.
- **Full Ed25519 byte re-verification at dispatch** — M3 (see D3).
- **Concurrent / multi-writer dispatch races** — the ledger's documented single-writer boundary (same as `findExistingPlanAdmitted`).

- [ ] **Step 3: Create the reader**

Create `packages/kernel/src/admitted-plan-reader.ts`:

```ts
import { existsSync } from "node:fs";

export interface AdmittedPlanRecord {
	readonly authorizedNextStep: string;
	readonly signedByKernel: boolean;
}

export interface AdmittedPlanReader {
	read(
		eventsDbPath: string,
		eventId: string,
	): Promise<AdmittedPlanRecord | undefined>;
}

/**
 * Reads a signed `plan_admitted` event by its tape event id and reports whether
 * a kernel-signed signature row exists. Structural signature check only (actor /
 * key / algorithm columns) — full Ed25519 byte verification is the external
 * verifier's job (M3). Read-only; never mutates the tape.
 */
export function createDefaultAdmittedPlanReader(): AdmittedPlanReader {
	return {
		async read(eventsDbPath, eventId) {
			if (!existsSync(eventsDbPath)) {
				return undefined;
			}
			const { DatabaseSync } = await import("node:sqlite");
			const db = new DatabaseSync(eventsDbPath, { readOnly: true });
			try {
				const row = db
					.prepare(
						"SELECT payload FROM events WHERE id = ? AND kind = 'plan_admitted'",
					)
					.get(eventId) as { payload?: string } | undefined;
				if (!row?.payload) {
					return undefined;
				}
				const parsed = JSON.parse(row.payload) as {
					PlanAdmittedV1?: { authorized_next_step?: string };
				};
				const sig = db
					.prepare(
						"SELECT actor_id, key_id, algorithm FROM event_signatures WHERE event_id = ?",
					)
					.get(eventId) as
					| { actor_id?: string; key_id?: string; algorithm?: string }
					| undefined;
				return {
					authorizedNextStep: parsed.PlanAdmittedV1?.authorized_next_step ?? "",
					signedByKernel:
						sig?.actor_id === "kernel" &&
```

```ts
import type { PlanForgePlan } from "./schema.js";

/**
 * Minimal packet shape PlanForge emits for dispatch. Structurally a subset of
 * `@buildplane/kernel`'s `UnitPacket`; planforge stays a zero-dependency leaf, so
 * the kernel type is not imported. The CLI re-validates each packet through
 * `parseUnitPacket` before handing it to the run loop.
 */
export interface DispatchedUnitPacket {
	readonly unit: {
		readonly id: string;
		readonly kind: string;
		readonly scope: string;
		readonly inputRefs: readonly string[];
		readonly expectedOutputs: readonly string[];
		readonly verificationContract: string;
		readonly policyProfile: string;
	};
	readonly execution: { readonly command: string };
	readonly verification: { readonly requiredOutputs: readonly string[] };
	readonly provenance_ref: string;
}

export interface DispatchPlanInput {
	readonly plan: PlanForgePlan;
	/** Tape event id of the signed `plan_admitted` authorizing this dispatch. */
	readonly admittedEventId: string;
	/** Policy profile each dispatched unit runs under. */
	readonly policyProfile: string;
}

/**
 * Build one packet per `PlanForgeTask` from an admitted plan. Each packet carries
 * `provenance_ref = admittedEventId`, the tape pointer the kernel admission gate
 * verifies. Packets are returned in plan order; the caller runs them respecting
 * `task.dependsOn`. S4 dispatches a deterministic no-op command (`true`) to prove
 * the admission → run → worktree path — real per-objective execution is S5.
 */
export function dispatchAdmittedPlan(
	input: DispatchPlanInput,
): DispatchedUnitPacket[] {
	const { plan, admittedEventId, policyProfile } = input;
	return plan.tasks.map((task) => ({
		unit: {
			id: `${plan.id}:${task.id}`,
			kind: "planforge-task",
			scope: task.workspace,
			inputRefs: [],
			expectedOutputs: [],
			verificationContract:
				task.verificationCommands.length > 0
					? task.verificationCommands.join(" && ")
					: "true",
			policyProfile,
		},
		execution: { command: "true" },
		verification: { requiredOutputs: [] },
		provenance_ref: admittedEventId,
	}));
}
```

```ts
const DEFAULT_DISPATCH_POLICY_PROFILE = "default";

/**
 * `buildplane planforge dispatch --input <file>`: dispatch an operator-admitted
 * plan as one run per PlanForgeTask. Fails closed (exit 1, `plan-not-admitted`)
 * when no signed plan_admitted exists on the tape. Tasks run sequentially so a
 * dependent task does not start if its predecessor fails.
 */
async function runPlanForgeDispatchCommand(
	args: readonly string[],
	cwd: string,
	stdout: (line: string) => void,
): Promise<number> {
	const inputPath = readFlag(args, "--input");
	if (!inputPath) {
		throw new Error(
			"Missing required --input <file> argument for PlanForge dispatch.",
		);
	}
	const jsonOut = args.includes("--json");

	const plan = createPlanForgeDryRunPlan(resolve(cwd, inputPath));
	const workspace = resolve(cwd);
	const runId = planAdmitRunId(plan.idempotencyKey);
	const admittedEventId = await findExistingPlanAdmitted(
		workspace,
		runId,
		plan.idempotencyKey,
	);
	if (!admittedEventId) {
		throw new Error(
			`plan-not-admitted: PlanForge plan ${plan.id} has no signed plan_admitted on the tape. Run \`buildplane planforge admit\` first.`,
		);
	}

	const packets = dispatchAdmittedPlan({
		plan,
		admittedEventId: String(admittedEventId),
		policyProfile: DEFAULT_DISPATCH_POLICY_PROFILE,
	});

	const { parseUnitPacket } = (await cliImport("@buildplane/kernel")) as {
		parseUnitPacket: (input: string) => unknown;
	};
	const { orchestrator, eventBus: cliEventBus } =
		await loadCliOrchestrator(workspace);

	const runs: Array<{ task: string; run_id: string; status: string }> = [];
	for (const packet of packets) {
		const result = await orchestrator.runPacketAsync(
			parseUnitPacket(JSON.stringify(packet)),
			cliEventBus,
		);
		runs.push({
			task: packet.unit.id,
			run_id: result.run.id,
			status: result.run.status,
		});
		if (result.run.status !== "passed") {
			break; // PF2 dependsOn PF1: stop the chain on first failure.
		}
	}

	const allPassed = runs.length === packets.length && runs.every((r) => r.status === "passed");
	stdout(
		jsonOut
			? formatJson({
					status: allPassed ? "dispatched" : "failed",
					plan_id: plan.id,
					admitted_event_id: String(admittedEventId),
					runs,
				})
			: `Dispatched PlanForge plan ${plan.id}: ${runs.length}/${packets.length} task(s).`,
	);
	return allPassed ? 0 : 1;
}
```

> `String(admittedEventId)` normalizes the tape `id` column (INTEGER or TEXT) to the string `provenance_ref`; the kernel gate's `WHERE id = ?` coerces it back. Confirm `dispatchAdmittedPlan` is added to the top-of-file `@buildplane/planforge` import.
